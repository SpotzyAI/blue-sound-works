# Language Settings

Configure your AI assistant to communicate in multiple languages.

## Supported Languages

Spotzy.ai supports 30+ languages:

### Major Languages

- **English**: US, UK, Australian, Indian accents
- **Spanish**: Spain and Latin American variants
- **French**: France and Canadian French
- **German**: Standard German
- **Portuguese**: Brazilian and European
- **Italian**: Standard Italian
- **Dutch**: Netherlands Dutch
- **Polish**: Standard Polish
- **Russian**: Standard Russian
- **Japanese**: Standard Japanese
- **Chinese**: Mandarin (Simplified and Traditional)
- **Korean**: Standard Korean
- **Arabic**: Modern Standard Arabic
- **Hindi**: Standard Hindi

### Additional Languages

Turkish, Swedish, Danish, Norwegian, Finnish, Czech, Ukrainian, Vietnamese, Thai, Indonesian, and more.

## Configuration

### Primary Language

Set default language:

1. Go to Assistant Settings
2. Select "Language"
3. Choose primary language
4. Select accent/dialect
5. Save changes

### Auto-Detection

Automatically detect caller language:

**Enable auto-detection:**
- Detects language in first few words
- Switches assistant language
- Confirms with caller
- Falls back to primary if uncertain

**Example:**
```
Caller: "Hola, necesito ayuda..."
Assistant: "¡Hola! Veo que prefieres español. ¿En qué puedo ayudarte?"
```

## Multi-Language Setup

### Single Language

Dedicated assistants:

- Create separate assistant per language
- Route by caller selection (IVR)
- Route by phone number
- Route by area code

### Multi-Language

One assistant, multiple languages:

**Advantages:**
- Unified knowledge base
- Consistent behavior
- Simpler management
- Cost-effective

**Configuration:**
```
Primary language: English
Secondary languages: Spanish, French
Auto-detection: Enabled
Fallback: Primary language
```

## Quality Considerations

### Accuracy by Language

Performance varies:

- **Tier 1** (Highest): English, Spanish, French, German
- **Tier 2** (High): Italian, Portuguese, Dutch, Polish
- **Tier 3** (Good): Most other languages

### Testing

Test each language:

- Native speaker review
- Accent understanding
- Cultural appropriateness
- Technical accuracy

## Best Practices

### Localization

Go beyond translation:

- Use local phrases
- Adapt examples
- Consider culture
- Use proper names

### Fallback Strategy

Handle language barriers:

```
If caller speaks unsupported language:
1. Attempt English
2. Offer language options
3. Transfer to human if needed
4. Collect callback info in any language
```

### Mixed Language

Handle code-switching:

```
Caller: "Hi, me gustaría información sobre..."
Assistant: "Hola! I can help you in Spanish or English. Which would you prefer? / ¿Cuál prefieres?"
```
